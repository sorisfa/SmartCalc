스마트 계산기 앱 개발 요청서 (Plan)

1. 목표

완전한 기능의 다기능 스마트 계산기 웹 애플리케이션 개발. 사용자가 일상생활에서 필요로 하는 다양한 계산 기능을 단일 인터페이스에서 직관적으로 제공합니다.

2. 핵심 기능 명세

  1. 기본 계산기: 사칙연산, 백분율(%), 괄호 연산 등 기본 수학 연산 및 초기화(C/AC) 기능
  2. 날짜 계산기: 특정 날짜 기준 D-day 계산, 기념일(100일, 1년 등) 계산, 두 날짜 사이의 일수 계산
  3. 환율 계산기: 한국 원(KRW), 미국 달러(USD), 일본 엔(JPY), 중국 위안(CNY) 등 주요 통화 간 변환 (실시간 환율 API 연동)
  4. 부동산 평수 계산기: 제곱미터(㎡) ↔ 평(坪) 양방향 자동 변환 기능
  5. 이자 및 대출 상환 계산기: 대출 금액, 금리, 기간을 입력받아 원리금균등, 원금균등, 만기일시상환 방식에 따른 월 상환액 및 총 이자 계산
  6. BMI 계산기: 신장과 체중 입력 시 체질량지수(BMI) 계산 및 비만도 상태 시각적 표시
  7. 연봉 및 실수령액 계산기: 연봉 입력 시 4대 보험 및 소득세 등을 공제한 예상 월 실수령액 계산
  8. 히스토리 기능: 각 계산기별 최근 계산 기록(History)을 로컬 스토리지에 저장하여 재확인 가능
  9. 반응형 디자인: 모바일 스마트폰, 태블릿, 데스크탑 웹 브라우저 환경에 모두 최적화된 유동적 레이아웃 제공

3. 기술 스택

  • 구조: HTML5 (시맨틱 마크업)
  • 스타일링: CSS3 (Vanilla CSS, Flexbox/Grid, CSS Variables, Media Queries)
  • 로직 및 동적 제어: Vanilla JavaScript (ES6+, Fetch API, LocalStorage)

4. 개발 마일스톤

  • Phase 1: 기획 및 디자인 명세(Plan.md, Design.md) 확정
  • Phase 2: 공통 UI 프레임워크 및 반응형 레이아웃 퍼블리싱
  • Phase 3: 핵심 계산기(기본, 부동산, BMI 등 로컬 연산) 로직 개발
  • Phase 4: 심화 계산기(환율 API 연동, 대출, 연봉 등) 및 히스토리 기능 구현
  • Phase 5: 크로스 브라우징 테스트, 디자인 폴리싱 및 최종 디버깅

스마트 계산기 디자인 요청서 (Design)

1. 디자인 목표 및 컨셉

  • 프리미엄 & 모던: 최신 웹 트렌드(Glassmorphism 혹은 세련된 Flat Design)를 반영하여 시각적으로 사용자에게 Wow(감탄)를 유도.
  • 직관적인 UX: 복잡한 계산 기능을 탭(Tab) 또는 사이드바 형태로 깔끔하게 분류하여 손쉽게 탐색.
  • 생동감 있는 인터랙션: 부드러운 마이크로 애니메이션과 호버 효과를 통해 조작감 극대화.

2. 반응형 레이아웃 정책

  • 모바일 (Mobile - 스마트폰)
    • 1단 세로 레이아웃(One-column) 적용.
    • 하단 네비게이션 바(Bottom Tab Bar) 또는 스와이프 가능한 상단 메뉴바 제공.
    • 키패드와 입력창이 화면을 꽉 채워 터치하기 쉽게 구성.
  • 태블릿 (Tablet)
    • 화면 분할 고려: 좌측 좁은 영역에 메뉴 배치, 우측 넓은 영역에 계산기 UI 표시.
    • 여유 있는 여백과 터치 친화적 버튼 크기 유지.
  • 데스크탑 (Desktop)
    • 대화면을 적극 활용한 다단(Multi-column) 레이아웃.
    • 사이드바 네비게이션 고정, 메인 영역에 계산기, 우측 패널에 계산 '히스토리' 동시 노출.

3. 색상 팔레트 (Color System)

  • Primary Color: 세련된 Vibrant Blue (#3B82F6) 또는 Teal (#14B8A6) - 주요 버튼 및 강조 영역
  • Background Color:
    • Light 모드: 깨끗한 Off-white (#F9FAFB) 및 은은한 그림자
    • Dark 모드: 고급스러운 딥 다크 그레이 (#111827 ~ #1F2937) (다크 모드 기본 지원 권장)
  • Text Color: 가독성 높은 짙은 회색 (#1F2937) 및 다크모드용 흰색 (#F9FAFB)
  • Accent/Error: 빨간색 (#EF4444)을 활용해 오류나 초기화(C/AC) 버튼 구별.

4. 타이포그래피 (Typography)

  • 기본 폰트: 'Inter', 'Roboto' 또는 'Pretendard' 등 가독성이 뛰어나고 숫자가 예쁘게 표현되는 산세리프 폰트.
  • 수치 표시: 계산 결과값은 볼드(Bold) 처리하고 크기를 대폭 키워 시선 집중 유도.

5. 컴포넌트 세부 디자인

  • 입력 폼 & 버튼: 부드러운 곡선(Border-radius: 12px~16px), 포커스 시 글로우(Glow) 효과 또는 외곽선 강조.
  • 트랜지션: 메뉴 이동, 결과값 출력, 히스토리 추가 시 ease-in-out 기반의 0.2~0.3초 애니메이션.
  • 결과 화면: 시인성을 높이기 위해 결과창의 배경색을 대비감 있게 설정.
